Building Routing Plans

Routing Plans are used to deliver claim information collected by call representatives. FNSNet places the claim information into a First Report of Injury claim form. This First report Of Injury claim form is sent via "transmissions" to destinations (A Destination is a place or person such as risk location, branch or caller.) by various methods (A call can be mailed, faxed or emailed.).

A Routing Plan (A Routing Plan is composed of a step or steps carried out by FNSNet to ensure the call reaches its destination.) is a plan for getting the call to its destination.

The Designer has two choices for building new routing plans. You can use Copy Routing Plan (Copying a routing plan makes a duplicate of a routing plan in the routing plan database.) to work from an existing model or use New Routing Plan (A new routing plan is an original routing plan added to the routing plan database.) to build a plan from scratch.

Building Routing Plans Using Copy Routing Plans

Building Routing Plans Using Copy Routing Plans

There are three steps to building routing plans using copy routing plans. You need to find the routing plan, copy the routing plan and paste the routing plan.

Find Routing Plan

You use "Routing Plan Search" to find the Routing Plan. Click Navigation Details For Routing Plan Search" for information on how to get to Routing Plan Search.

Navigation Details For Routing Plan Search

1. From the navigation bar, choose "Maintenance" and click the "+" sign next to the "Routing Plan Related" on the Maintenance tree.

2. Click "Routing Plans" to display the Routing Plans Search page.

Enter or select the Routing Plan Search information.

  • Select Destination Type from drop down list.

  • Select Input System Name from drop down list.

  • Select Line of Business from drop down list.

  • Select Enabled or Enable from drop down list.

  • Enter Routing Plan ID and Description.

  • Select Type of Search

The default is "Begins With". "Begins With" means Designer returns results beginning with the criteria you entered. ""Contains" means Designer returns results containing the criteria you entered. "Exact" means Designer returns results with the exact criteria you entered.

  • Click the "Search" button.

  • Highlight the result from the results list and click the "Select" button.

Example

  • Routing Plan appears.

Example

Copy Routing Plan

You need to copy the routing plan to the windows clipboard.

  • Right click the mouse on the routing plan in the Routing Plan Details area and click "Copy Routing Plan" on the popup menu.

 

Paste Routing Plan

You need to paste the routing plan from the Business Entity Summary page.

From the navigation bar, choose "Hierarchy" and click the "+" sign next to FNS in the Hierarchy tree. Select the client in the Hierarchy Tree to go to the Business Entity Summary page in the right frame and click the "+" sign next to Routing Plans on the Business Entity page.

  • Click "Paste" on the Routing Plans bar.

Notes

Refresh

The Refresh icon updates the Designer with the latest information. It appears in the Call Flow Frames section.

 

Attach and Detach Rule to Routing Plan

Attach Rule

If you don't attach a rule, the routing plan will always be enabled. You can only attach a "Enable" rule at the routing plan level.

Click the Attach Rule icon in the Routing Plan section to attach a rule to the routing plan. Enter the Rule Search information below.

  • Select Type Of Rule from drop down list.

  • Enter Rule Text and Rule ID.

  • Select Type Of Search

The default is "Contains". "Begins With" means Designer returns results beginning with the criteria you entered. ""Contains" means Designer returns results containing the criteria you entered. "Exact" means Designer returns results with the exact criteria you entered.

  • Click the Search button.

  • Select Result from results list.

  • Click the Select button.

  • Click the Save button.

 

De-Attach Rule

Click the De-Attach Rule icon in the Routing Plan section to detach the rule from the routing plan. Click the Save button.

Building Routing Plans Using New Routing Plans

Building Routing Plans Using New Routing Plans

New Routing Plans enables you to create a new routing plan from scratch.

Click Navigation Details for information on how to get to Building Routing Plans Using New Routing Plans.

Navigation Details

1. From the Navigation bar, choose Hierarchy.

2. Click the "+" sign next to FNS in the Hierarchy tree.

3. Select the client in the Hierarchy Tree to go to the Business Entity Summary page in the right frame.

4. Click the "+" sign next to the Routing Plans on the Business Entity page.

Click "New" button on the Routing Plans bar to go to the New Routing Plan page.

Enter or select Routing Plan information in the Routing Plan Summary area.

  • Enter Description.

  • Select Destination Type from drop down list.

  • Select Input System Name from drop down list.

  • AH Step ID is already filled in.

  • Select line of business and State from drop down list.

  • Click the "Save" button.

Example

In the Routing Plan Summary area the new routing plan appears with a name and number.

 

Notes
