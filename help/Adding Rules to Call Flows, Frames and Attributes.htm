Adding Rules to Call Flows, Frames and Attributes

You have the option of using rules with call flows, frames and attributes. If you want your call flows, frames and attributes to function in certain situations, you should use rules (A rule is a formula that makes something happen in certain situations.). For example if you want a particular frame enabled in certain situations you would attach a rule to the frame. If you wanted an attribute visible in certain situations you would attach a rule to the attribute.

Adding and Removing Rules to Call Flows

Adding and Removing Rules to Call Flows explains attaching rules to call flows, creating call flow rules and detaching call flow rules.

You have two choices when adding rules to call flows. You can either attach rule to call flow or create a call flow rule. Whether you want to attach a call flow rule or create a call flow rule depends on how you want to use the rule. If you can use a call flow rule as is, you may want to attach rule to call flow (An attached rule is an existing rule connected or joined to a call flow.). If you plan on making changes to the call flow rule, you may want to create a call flow rule (Creates a new or original call flow rule.). If you change your mind and you want to remove the rule from the call flow, you may want to detach call flow rule (A detached call flow rule is disconnected from the call flow.).

Adding and Removing Rules to Frames

Adding and Removing Rules to Frames explains attaching rules to frames, creating frame rules and detaching frame rules.

You have two choices when adding rules to frames. You can either attach rule to frame or create a frame rule. Whether you want to attach a frame rule or create a frame rule depends on how you want to use the rule. If you can use a frame rule as is, you may want to attach rule to frame (An attached rule is connected or joined to a frame.). If you plan on making changes to the frame rule, you may want to create a frame rule (Creates a new or original frame rule.). If you change your mind and you want to remove the rule from the frame, you may want to detach frame rule (A detached frame rule is disconnected from the frame.).

Adding and Removing Rules To Attributes

Adding and Removing Rules to Attributes explains attaching rules, creating rules and detaching rules to Instance Properties and Attribute Properties.

Whether you want to use Instance Properties or Attribute Properties depends on how you want to use the properties of the attribute. If you plan on making changes to the properties, you may want to use Instance Properties (An Instance Properties is essentially a copy or duplicate of attribute properties. If you change Instance Properties these changes effect only your "duplicate" of the properties.).

Instance Properties allows you greater flexibility because you are using a duplicate or copy of the attribute's properties. This allows you to modify the copy of the attribute properties to suit your needs. Therefore you should use Instance Properties if you are going to modify properties.

If you plan on using the Attribute Properties as is, you may want to use Attribute Properties (An Attribute Properties are the properties or characteristics of an attribute. If you change Attribute Properties changes directly effect the attribute which is often shared.).

Attribute Properties has important consequences because you are using the actual properties of an attribute. If you make changes to the Attribute Properties you are changing the properties of the attribute in the database. This effects all users of the attribute. Therefore you should use Attribute Properties only if you do not want to change properties.

You have two choices when adding rules to Instance Properties and Attribute Properties. You can either attach rule to the Instance Properties or Attribute Properties or create a rule for Instance Properties and Attribute Properties. Whether you want to attach a attribute rule or create a attribute rule for an Instance Properties or Attribute Properties depends on how you want to use the rule. If you can use a attribute rule as is, you may want to attach rule to Instance Properties or Attribute Properties (An attached rule is connected or joined to a Instance Properties or Attribute Properties.). If you plan on making changes to the attribute rule, you may want to create a attribute rule (Creates a new or original attribute rule.) for the Instance Properties or Attribute Properties. If you change your mind and you want to remove the rule from Instance Properties or Attribute Properties, you may want to detach attribute rule (A detached rule is disconnected from the attribute.).

Attaching Rules to Call Flows

Click the "x" button in the upper right corner of Internet Explorer to close the Call Flow Editor and go to the Call Flow page.

Click the Attach Rule icon in the Account Call Flow section on the Call Flow page.

Enter or select Rule Search information.

  • Select "Call Flow" for type of search.

  • Enter Rule Text and Rule ID.

  • Select Search Type.

The default is "Contains" meaning Designer returns information containing the criteria you entered. "Contains" is helpful when you do not know the exact information you want. "Begins With" means Designer returns results beginning with the criteria you entered. "Begins With" is useful for returning all information in a table. "Exact" means Designer returns the exact criteria you entered. "Exact" is helpful when you know the exact information you want.

  • Highlight the call flow rule from the results list and click "Select" on the Rule Maintenance dialog box.

Rule Types

Rule Types

There are several rule types used in the Designer.

  • An Enabled Rule allows the editing of a call flow, frame or attribute.

  • A Valid Rule allows the editing of a call flow, frame or attribute, but verifies the data entered.

  • A Visible Rule shows a call flow, frame or attribute, but does not allow editing.

  • A Persist Rule saves data to the database in a call flow, frame or attribute.

  • An Action Rule triggers and event in a call flow, frame or attribute.
