Account Hierarchy

The Designer uses an Account Hierarchy Tree for navigating and displaying information. In the left frame, information is structured like a tree with branches expanding to show Clients, Risk Locations and Accounts.

The Account Hierarchy Tree contains features designed to make using the tree easy. You can expand or collapse client or account information. In addition you can Show As Top Node, Add To Favorites, Add Child Node, Node Search, Filter Properties and Retrieve Nodes.

Expand Or Collapse Account Information

Expand or collapse a branch to see only the information you want to see.

Show As Top Node

Show As Top Node moves the client, risk location or account to the top of the Hierarchy Tree.

Add To Favorites

Add To Favorites copies a client, risk location or account to "Favorites" located underneath the Hierarchy Tree. This gives you fast access to a particular client, risk location or account without having to find the information on the Hierarchy Tree.

Add Child Node

Add Child Node adds a node to a parent node. This enables you to add clients, risk locations and accounts.

Node Search

Node Search enables you to lookup information in the Hierarchy Tree. You use Node Search to find a particular business entity, call flow, routing plan or policy.

Filter Properties

Filter Properties applies a filter to the Hierarchy Tree. This creates a custom view on the Hierarchy Tree enabling you to find information quickly. The filter blocks unwanted information and displays on the information you requested in the Hierarchy Tree.

Retrieve Nodes

Retrieve Nodes gets nodes that were not displayed under a parent node due to the number of nodes.
